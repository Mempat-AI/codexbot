# Security Policy

## Reporting a Vulnerability

If you believe you have found a security vulnerability, do not open a public issue.

Instead:
- contact the maintainers through a private channel if one is available
- or use GitHub private vulnerability reporting if it is enabled for this repository

Please include:
- a description of the issue
- steps to reproduce it
- impact assessment
- any suggested mitigation if you have one

## Disclosure Expectations

Please allow time for investigation and remediation before public disclosure.

## Scope

This project is under active development. Security fixes will be handled on a best-effort basis, but responsibly reported issues that affect current supported code will be reviewed.
